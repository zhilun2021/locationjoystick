Settings

This is where you set movement speeds, adjust how realistic the simulated GPS signal looks, choose which controls appear in the widget, set roaming defaults, and back up or restore your data. Settings is split into four screens: Movement & GPS, Menus, Favorites & Routes, and Roaming.

  • Movement & GPS lets you edit the Slow Walk, Walk, Run, Bike, and Drive speed presets used by the joystick, routes, and roaming, and turn GPS realism features on or off one by one: bearing hold, altitude drift, warm-up, satellite count, and signal dropouts.
  • Menus lets you choose which features appear in the floating widget and on the map, and drag to reorder them.
  • Favorites & Routes lets you turn on curated locations and routes for quick access.
  • Roaming lets you set the default area, distance, speed, road-following, and auto-return behavior for roaming sessions.

Movement & GPS

This screen combines speed settings with GPS realism controls. Set your movement speeds here, and adjust how the simulated signal behaves.

Speed profiles

Five presets control movement speed for the joystick, route playback, and roaming. Edit any of them, and the change applies right away. A km/h ↔ mph switch at the top converts every speed shown in the app at once.

ProfileDefaultPace
Slow Walk 1.1 km/h Strolling, browsing shopfronts
Walk 2.0 km/h Regular walking pace
Run 8.0 km/h Jogging, follows footpaths when road-following is on
Bike 15.0 km/h Cycling, follows bike lanes when road-following is on
Drive 54.0 km/h Driving, well above the anti-cheat threshold below

There's no upper limit, so you can type any speed you want. The only floor is 0.04 km/h: anything entered below that gets raised to it automatically, since a true zero would mean the simulated device never moves at all.

Each profile also has a checkbox that controls whether the widget's Speed cycle button includes it. Walk, Run, and Bike are on by default, while Slow Walk and Drive are off unless you turn them on. This only affects cycling: all five speeds stay editable here no matter what.

Anti-cheat warning

Set any profile above 28.8 km/h (17.9 mph) and a warning appears inline below that field:

"Speed exceeds 8 m/s, may trigger anti-cheat in some games"

It's just a heads-up: nothing gets blocked, capped, or sent anywhere. The threshold sits just above any realistic human walking or running pace, which is what many location-aware games watch for. Each profile is checked on its own, so lower it below the threshold to clear the warning, or ignore it if that's not something you need to worry about.

Location randomness

Adds small random variation to each location update, mimicking the natural drift of a real GPS chip. Set any value to 0 to turn off that particular variation.

OptionAppliesWhat it does
Idle radius Stationary The largest random offset while you're standing still. 1 to 3 m looks like a real chip drifting slightly in place.
Idle interval Stationary How many seconds pass between new offsets while you're still. A shorter interval makes the trace look jitterier.
Moving radius Moving The largest random offset while moving. Keep it below the idle radius to avoid unnatural-looking jumps.
Moving interval Moving How many seconds pass between new offsets while moving.
Idle speed variation Stationary A random percentage added to or subtracted from the reported speed at rest. 5 to 10% mimics normal measurement noise, while 0 reads as perfectly still.
Moving speed variation Moving A random percentage added to or subtracted from the reported speed while moving, so it doesn't look like a flat, constant number.

GPS realism

These toggles work independently to make the spoofed GPS signal behave more like a real hardware chip. Some apps and games check these details to spot fake locations, so turning on all the options lowers the chance of being detected.

OptionDefaultWhat it does
Hold bearing when stationary On Keeps reporting your last direction instead of snapping back to north when you stop. A sudden reset to north is a common giveaway.
Vary altitude On Drifts the reported altitude slightly rather than holding one constant height.
GPS warm-up simulation Off Starts each session with lower accuracy that sharpens over about 30 seconds, like a phone getting its first GPS fix.
Realistic satellite count On Reports 7 to 14 visible satellites, with 6 to 12 actually in use. Zero satellites is a giveaway some apps check for.
Suspended mocking Off Briefly stops sending updates, about 2 seconds out of every 10, to mimic real signal dropouts. Turns off automatically during route playback.

Map

Controls how the map inside the app behaves while spoofing is active.

OptionDescription
Remember last location Brings back your most recent spoofed position when you reopen the app, so you don't have to re-enter coordinates. Your position is saved on your device after every update.
Follow location on map Keeps the map centered on your spoofed position as it moves. Panning the map by hand turns this off until you tap the re-center button.

Menus

Appearance

A Light mode switch lets you flip the whole app to a bright, high-contrast look instead of the default dark orange-on-black theme, handy for reading the screen outdoors in bright sunlight. The change applies immediately, everywhere in the app.

This is where you choose which quick-access features appear in the floating widget and on the map screen, and in what order. Each feature has its own checkbox for the widget and for the map, so you can turn either on independently, plus a drag handle on the left so you can reorder the whole list. Both surfaces follow the same order, so reordering once keeps everything consistent between them.

FeatureIconDescription
Map shortcut 📍 Only available in the widget. Opens a compact map that fills the screen without switching back to the main app. Supports walk-to, teleport, and roaming controls right there.
Show/hide joystick 👁 Only available in the widget. Turns the floating joystick on or off without opening the app. The joystick needs to already be running from the main screen.
Lock joystick 🔒 Only available in the widget. Holds the joystick's direction after you let go, so the simulated device keeps moving without you touching the screen. Tap again to unlock.
Favorites ❤️ Available in both the widget and the map. Lists your saved favorite locations with one-tap teleport and walk shortcuts. From the widget, you can also save your current spoofed position as a new favorite.
Routes 🗺 Available in both the widget and the map. Lists all your saved routes, with options to start, loop, reverse, or return them. While a route is playing, the button expands to show pause and stop controls.
Roaming 🧭 Only available on the map. Opens the roaming setup screen so you can configure and start a random walk.
Search 🔍 Only available on the map. Opens the search bar so you can find and jump to a place by name.
Speed cycle ⚡ Only available in the widget. Cycles through your enabled speed profiles with a single tap; you choose which ones below. The icon changes to show the active profile.

Starting and stopping location simulation is always available from the bar at the top of every screen, so it isn't one of the configurable features above.

Roaming

Set the default area, distance, speed, road-following, and auto-return behavior for roaming sessions.

OptionDescription
Radius The farthest distance from the starting point that random waypoints can land. Larger values spread movement further from the center.
Route distance The total distance to walk before the roaming session ends. It stops once the distance walked adds up to this value.
Speed profile Which speed preset, Walk, Run, or Bike, is used for roaming movement. Bike also switches to the cycling option for road-following routes.
Follow roads Routes each leg through a road-mapping service so movement follows real walkable paths instead of straight lines. Falls back to a straight line if that service isn't available.
Return to start After the main roaming loop completes, the simulated device walks back to the original starting position.

External services

locationjoystick works offline by default. It doesn't need an account, and it doesn't send any usage data or tracking information anywhere. That said, two optional features do reach out over the network:

ServiceWhen usedWhat is sent
OSRM
router.project-osrm.org		 Road-following routes, roaming with Follow roads turned on, and guided route creation. This service is only contacted when you specifically choose road-following. A list of latitude and longitude pairs, your waypoints or random roaming targets. No device identifiers and no account data are sent. The public demo server used for this is run by the OSRM project.
Nominatim
nominatim.openstreetmap.org		 Place-name search in the route creator and the favorites map picker. Only contacted when you type a search query. Whatever text you type into the search box, a place name or address. No GPS coordinates and no identifiers are sent. The public search service used for this is run by the OpenStreetMap Foundation.

Map images come from OpenStreetMap and are fetched from tile.openstreetmap.org as you pan and zoom the map. Each request includes only the map tile's position, its zoom level and grid coordinates, and no personal data.

All three services are contacted only when the feature that needs them is in use, and you can avoid all of them entirely by not using road-following, place search, or the map itself, for example by entering coordinates directly into Favorites.

QR transfer

Share your whole configuration between two devices by scanning a single QR code, without a cable, an account, or any cloud service involved.

  • Both devices need to be on the same Wi-Fi network.
  • The sending device shows a QR code, plus a short backup code. The receiving device scans the code, or types it in, and imports right away.
  • No cloud is involved. The data travels directly between the two devices over the network.

Export / import

Back up or restore all of your app data: routes, favorites, speed profiles, widget setup, location randomness settings, and roaming defaults.

ActionDescription
Export via QR code Shows a QR code that another device on the same Wi-Fi network can scan to receive your current configuration.
Export settings Saves a backup file to your device, which you can share or keep as you like.
Import from QR code Opens the camera to scan a code shown on another device on the same Wi-Fi network, then asks whether to add it to your existing data or replace everything.
Import via code Type in the short code shown on the sending device instead of scanning. Both devices still need to be on the same Wi-Fi network.
Import from file Opens a file picker so you can choose a previously exported backup file. Asks whether to add it or replace everything.
Import from GPS Joystick Migrates routes and favorites from a GPS Joystick export file. Imported routes are saved as straight-line segments.
Import from YAMLA Migrates favorites and speed settings from a YAMLA export file.

Next to the export and import buttons is a Reset all data button. It asks you to confirm first, then permanently deletes all your favorites, routes, and settings, without you needing to clear the app's data through your phone's own settings. Your permission grants stay in place, so you won't have to go through setup again.
